Overview

You can install and run Analytical Engine on your local computer or set it up in a server mode (for details see Server Mode section).

Analytical Engine * works with self-contained local Spark; * can connect to any external Spark cluster (YARN, Mesos or Spark Standalone); * works on multiple platforms, including Windows, OS X and Linux.

If you want to install Analytical Engine on Linux you should use Dockerized Analytical Engine.

If you want to install Analytical Engine on OS X or Windows you should use Analytical Engine as a Vagrant Image.

You can also build Analytical Engine from source code. Follow instructions at Analytical Engine github repo .

Dockerized Analytical Engine

Works on: Linux

Installation

This method of deployment uses a set of Docker containers and Docker Compose to connect them with each other. Docker is a lightweight, secure containerization tool that allows installing a complex application along with its dependencies without cluttering your OS.

Prerequisites

  • Docker (tested on versions 1.10.3, 1.11.2 and 1.12.0)
    • Docker Toolbox and Docker on Mac are not supported.
  • Docker Compose (tested on versions 1.7.0 and 1.7.1)
  • Google Chrome (version >= 51) or Mozilla Firefox (version >= 48)

Running Analytical Engine

  1. Download the docker-compose.yml file, which serves as a config for Docker Compose (this may take a few minutes).
  2. Run docker-compose up in the directory containing the downloaded file.
  3. When Analytical Engine is ready, open it in your browser. It is accessible at http://localhost:33321.

Troubleshooting

  • In case of unexpected errors during Analytical Engine usage, try rebooting Dockerized Analytical Engine by running:

      docker-compose down docker-compose up

  • Bug reports and all other feedback can be sent using the Feedback option in the Analytical Engine Editor in the upper-left corner.

Conflicting Ports

Analytical Engine, by default listens on port 33321. In rare cases this configuration may conflict with services running on your computer. This can be easily remedied by replacing 33321 with a port of your choosing in docker-compose.yml file. After restarting Analytical Engine, it will be available under the address with the new port.


Analytical Engine as a Vagrant Image

Works on: OS X, Windows, Linux

Installation

On some operating systems Docker is not supported natively. That’s why we also distribute Analytical Engine as a Vagrant image.

Prerequisites

  • Vagrant (tested on version 1.8.1)
  • VirtualBox (tested on version 5.0.10, there is an issue with version 5.0.12)
  • Google Chrome (version >= 51) or Mozilla Firefox (version >= 48)

Running Analytical Engine

  1. Download the Vagrantfile that will allow you to spin up Analytical Engine's virtual machine.
  2. Run vagrant up in the directory containing Vagrantfile. This may take a few minutes.
  3. When Analytical Engine is ready, go to http://localhost:33321 in your browser.

Troubleshooting

  • If you cannot start the Analytical Engine virtual machine, please check if virtualization is enabled in BIOS.

  • In case of unexpected errors during Analytical Engine usage, try rebooting Analytical Engine by running:

      vagrant reload
  • Bug reports and all other feedback can be sent using the Feedback option in the Analytical Engine Editor in the upper-left corner.

Conflicting Ports

Analytical Engine, by default listens on port 33321. In rare cases this configuration may conflict with services running on your computer. This can be easily remedied by replacing 33321 with a port of your choosing in Vagrantfile. After restarting Analytical Engine, it will be available under the address with the new port.

Notes for Windows Users

Problem with Special Characters in HOMEPATH Environment Variable

By default, Vagrant keeps boxes and configuration in %HOMEPATH%/.vagrant.d, but cannot access it when a %HOMEPATH% includes non-ASCII characters. It is a well known Vagrant bug and results in:

> vagrant up
...
Failed writing body (0 != 16383)

There are at least two ways to overcome the problem:

  • Change the user name to one with ASCII characters only

  • Change the Vagrant home directory to one with ASCII characters only:

      setx VAGRANT_HOME c:\.vagrant.d


Problem with Downloading Vagrant Box

Vagrant fails to download Analytical Engine Box file, but does not print any specific error message.

An error occurred while downloading the remote file. The error
message, if any, is reproduced below. Please fix this error and try
again.

To overcome this issue, install Microsoft Visual C++ 2010 SP1 Redistributable Package (x86).
